You may be wondering why we are diving into another linear algebra library considering we have already studied numpy.linalg. Simply put, scipy.linalg contains everything that numpy.linalg does but offers additional functionality. It is used in conjuction with numpy ndarrays.

Linear Systems#

b = np.array([[6],[12],[18]])

x = linalg.solve(A, b)

Sparse Matrix#

A sparse matrix (loosely defined) is a matrix with a many zero elements. When dealing with large sparse matrices it is extrememly useful to take advantage of their sparseness and the memory requirements, speeding-up our code. To convert from a memory dense matrix to a memory sparse matrix we use scipy.sparse.csr_matrix(A).

A = np.array([[0,0,1],[2,0,0],[0,3,0]])
print(A,'\n')

A = scipy.sparse.csr_matrix(A)
print(type(A),'\n')

Matrix Exponential#

The matrix exponential of a matrix \(A\) is written as

\[ \exp (A) = e^{A} = \sum_{k=0}^{\infty} \frac{1}{k!}A^k . \]

This seems like a monumental task but it can easily be achieved with scipy.linalg.expm(A).

n=4
A = (1/np.sqrt(n))*linalg.hadamard(n)
print(A,'\n')

expA = linalg.expm(A)

print(expA,'\n')

Also we can verify that $\( \exp \left(A^T\right) = \exp \left(A\right)^T \)\( and \)\( \exp \left(A^\dagger \right) = \exp \left(A \right)^\dagger \)$ up to a computational error.

A = rnd.random((n,n)) + 1j*rnd.random((n,n))

One key point with matrices is that if \(A\) and \(B\) do not commute, $\(AB \neq BA \iff [A,B]= AB -BA \neq0.\)\( then \)\( e^A e^B \neq e^{A+B}.\)$

But if $\([A,B]= AB -BA =0\)\( then \)\(e^Ae^B = e^{A+B}. \)$

A = np.array([[0,1],[0,0]])
B = np.eye(2)
# These matrices commute

EA = linalg.expm(A)@linalg.expm(B) == linalg.expm(A+B)
print(EA,'\n')

A = np.array([[0,1],[0,0]])
B = np.array([[0,1],[1,0]])
# These matrices do not commute

EA = linalg.expm(A)@linalg.expm(B) == linalg.expm(A+B)
print(EA,'\n')

Unitary Matrix#

A Unitary Matrix is defined as a matrix that satisfies the following equation, $\(AA^{\dagger} = A^{\dagger}A = \mathbb{1}\)\( where \)\mathbb{1}\( is the identity matrix. In other words the Hermitian conjugate of the matrix is it's multiplicative inverse, \)\(A^{-1} = A^\dagger\)$ The Hadamard gate is unitary as we see below.

n=4
A = (1/np.sqrt(n))*linalg.hadamard(n)
print(A,'\n')
A = np.matrix(A)

print(A*A.H==A.H*A,'\n')

print(A*A.H,'\n')

Tensor Product#

In the function scipy.sparse.kron(A,B) the matrices/vectors A and B can be sparse or dense. The tensor product is computed as a sparse matrix. The tensor product of two \(2\times2\) matrices \(A\) and \(B\) is as follows

\[\begin{split} A = \begin{pmatrix} a_{11} & a_{12} \\ a_{21} & a_{22} \end{pmatrix}, \ \ \ \ B = \begin{pmatrix} b_{11} & b_{12} \\ b_{21} & b_{22} \end{pmatrix},\end{split}\]
\[\begin{split} A \otimes B = \begin{pmatrix} a_{11} B & a_{12} B \\ a_{21} B & a_{22} B \end{pmatrix} = \begin{pmatrix} a_{11} \begin{pmatrix} b_{11} & b_{12} \\ b_{21} & b_{22} \end{pmatrix} & a_{12} \begin{pmatrix} b_{11} & b_{12} \\ b_{21} & b_{22} \end{pmatrix} \\ a_{21} \begin{pmatrix} b_{11} & b_{12} \\ b_{21} & b_{22} \end{pmatrix} & a_{22} \begin{pmatrix} b_{11} & b_{12} \\ b_{21} & b_{22} \end{pmatrix} \end{pmatrix} = \begin{pmatrix} a_{11} b_{11} & a_{11} b_{12} & a_{12} b_{11} & a_{12} b_{12} \\ a_{11} b_{21} & a_{11} b_{22} & a_{12} b_{21} & a_{12} b_{22} \\ a_{21} b_{11} & a_{21} b_{12} & a_{22} b_{11} & a_{22} b_{12} \\ a_{21} b_{21} & a_{21} b_{22} & a_{22} b_{21} & a_{22} b_{22} \end{pmatrix}.\end{split}\]

The sucture is similar for differently shaped matrices and vectors.

A = np.array([[1, -2], [-1, 2]])
B = np.array([[1, 2], [3, 4]])

result_dense = scipy.sparse.kron(A,B)

print('Result dense = \n',result_dense.todense(),'\n')

A = scipy.sparse.csr_matrix(A)
B = scipy.sparse.csr_matrix(B)

result_sparse = scipy.sparse.kron(A,B)

print('Result sparse = \n',result_sparse.todense())

Least Square Fit#

The package scipy.linalg also supports least-square fitting, a method that fits data points to a curve. The relavent function is scipy.linalg.lstsq(A, c). To demonstrate how this works, we are going to use this function to approximate the values of \(c_1\) and \(c_2\) in the following function $\( y = c_1 x^2 + c_2 x.\)\( We will generate data points by adding noise to this function for certain values of \)c_1\( and \)c_2\(. We will then then try to fit the noisy data with `scipy.linalg.lstsq(A, c)`, comparing our approximate values of \)c_i$ to the actual values.

Lets set $\( c_1 = 0.5\)\( and \)\( c_2 = 2.0\)\( \)A\( is a matrix of the set of functions, of \)x\(, describing \)y\(. So in this case \)\( A = [x^2,x]\)$

c1, c2 = 0.5, 2.0

x = np.arange(11)

y = c1*x**2 + c2*x

noise = y + 0.2 * np.max(y) * rnd.random(len(y))

A = np.c_[x**2,x]

c, resid, rank, sigma = linalg.lstsq(A, noise)

y2 = c[0]*x**2 + c[1]*x

fig = plt.figure()
ax = fig.add_axes([.1,.1,.8,.8])

ax.plot(x,noise,'x',x,y2)
ax.set_xlabel('$x$')
ax.set_ylabel('$y$')

plt.show()

Singular Value Decomposition#

Singular value decomposition is an extension of the eigenvalue, eigenvector problem to non-square matrices. The following matrices, and vector, are generated using scipy.linalg.svd() $\( A = USv, \)\( where \)A\( is a non-square matrix, \)U\( is a square matrix, \)S\( is a diagonal matrix, and \)v$ is a vector.

A = rnd.random((3,2))
print('A = ',A,'\n')

U, D, v = linalg.svd(A)

print('U = ',U,'\n')
print('Diagonal Elements = ',D,'\n')
print('v = ',v,'\n')

S = scipy.sparse.lil_matrix((3,2))
S[0,0] = D[0]
S[1,1] = D[1]

S = S.todense()

Schur Decomposition#

The Schur Decomposition of \(n\times n\) matrices gives said matices in the form $\(A = Z T Z^\dagger \)$

Where \(Z\) is a unitary matrix, and \(T\) is an upper-triangular(or quasi-upper) matrix. We can request these matrices in a real or complex form using scipy.linalg.schur(A) and scipy.linalg.schur(A,'complex') or by using scipy.linalg.rsf2csf(A) for the complex form.

A = rnd.random((3,3))
print('A = ', A, '\n')

T, Z = linalg.schur(A)
Tc, Zc = linalg.schur(A, 'complex')

T, Z, Tc, Zc = map(np.matrix,(T,Z, Tc, Zc))

print('A = ',Z*T*Z.H, '\n')

print('A = ',Zc*Tc*Zc.H)
